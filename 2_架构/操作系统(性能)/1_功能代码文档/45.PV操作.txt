临界资源：只能以互斥的形式访问  
临界代码区：通过某一块代码区来替代的资源
	 （原子性：要么一次都不执行，要么都执行）

同步与互斥：
	互斥：每一时刻，只能有一个线程或者进程执行，其他线程或进程等待。
	同步：为了共同完成某任务而规定的一种先后次序。

模型:
				
	1.读写者模式:互斥	
		
	2.生产者与消费者：同步，互斥


经典模型：一个生产者与一个消费者 
	1、一个生产者（厂商）  
	2、一个消费者（买家）
	3、公有一个缓冲区（仓库）：最大的容量20辆
	
	生产者-----放东西-----》缓冲区《-----取东西---- 消费者	
	
	PS：1、先生产后消费
	    2、当缓冲区为满时，生产者必须等待消M者消M
	       当缓冲区为空时，消费者必须等待生产者生产
		   
	Ps:锁，做不到，因为当不为满且不为空时，生产者和消费则应该都可以执行

	伪代码：
	
	P:	empty(仓库的空间(用来放货的)数量)
		
	V:  full(仓库中货物的数量)

	生产者 producer：
		
		while(true)
		{
			
			P(empty);     空间-1
			开始生产，并将货物保存到仓库中（容器）
			V(full);      货物+1
		}
	消费者 consumer：
		while(true)
		{
			P(full);     货物-1
			从仓库中取出货物（容器）
			V(empty);    空间+1
		}
		
		
多个生产者与多个消费者：
	1、一组生产者（厂商）  
	2、一组消费者（买家）
	3、公有一个缓冲区（仓库）：最大的容量20辆
	PS：	
	   同步：
		1、先生产后消费
		2、当缓冲区为满时，生产者必须等待消费
	       当缓冲区为空时，消费者必须等待生产	
	   互斥：（多个使用者使用同一块空间时，如果这个空间时一块公共资源，必须互斥）
	   	1、各个生产者之间，各个消费者之间存在互斥地访问

	   伪代码：
	   	empty=5--缓冲区空间的数量	（生产者  占用 空间）
		full --缓冲区元素的数量		（消费者  消M 货物）
		
		mutex1---生产者之间
		mutex2---消费者之间
	生产者1-10：
		
		while(true)
		{
			
			P(empty);     空间-1
			lock(mutext1)//上锁--互斥地访问空间  Ps：将锁放在上面(有5个生产者，同时生产)，放在上面又是一个一个生产  --》错误
			开始生产，并将货物保存到仓库中（容器）
			unlock(mutex1);
			V(full);      货物+1
		}
	消费者1-n：  queue q
		while(true)
		{
			p(full) //货物-1
				lock(mutex2)//以互斥形式访问
				从buffer中取出产品
				unlock(mutex2);
			v(empty);//空间+1
		}
		
		
生产者和消费者的优势:
	
1.解耦：
	耦合(就是所谓的依赖)
2.支持并发 --》（消费者方法返回之前，生产者一直等待，浪费时间，优势体现出来了））
3.支持忙闲(生产者快，消费者慢)不均
